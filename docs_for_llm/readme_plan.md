# 核聚变门户网站项目简介（Project Overview）

本项目致力于构建一个面向公众、工程实践者与科研人员的 **核聚变科学与工程知识门户（Fusion Portal）**。
它不仅是一个网站，而是一个 **可持续演进的知识平台、内容管理系统**，以及 **未来 AI 科研助手的基础设施层**。

Portal 将整合核聚变的科学原理、工程建设、产业化进展、重大事件、政策环境、科研论文与行业应用，并逐步具备智能辅助能力，成为核聚变领域的数字入口。

## 🎯 一、项目目标（Project Goals）

### 1. 面向公众
提供清晰、系统、可信的核聚变科普资料、时间线、技术路线图、行业动态以及可视化展示。

### 2. 面向工程师与高校/科研机构
提供工程文档索引、论文索引、行业资讯、产业链生态等结构化数据。

### 3. 面向未来 AI 生态
构建可供 AI 检索、分析、生成内容的结构化知识体系，为未来的聚变研究数字化、仿真展示、AI 助手等能力铺设底层基础。

> **简言之：**
> 这是一个能持续生长的**核聚变知识基础设施（Fusion Knowledge Infrastructure）**，而不是一次性的内容项目。

## 🧩 二、当前架构（Current Architecture）

本项目使用 **前后端分离（Next.js + Express）** 的模式运行，并预留未来迁移至 CMS 与 Next.js Route Handlers 的演进路径。

### 前端（frontend/）

*   **技术栈：** Next.js 14（App Router）+ TypeScript + Tailwind CSS
*   **站点结构（迁移后正式结构）：**
    *   `/`：门户首页（时间线与聚变概述）
    *   `/history`：核聚变发展历史（SWR 无限滚动）
    *   `/links`：聚变相关资源索引
    *   `/papers`：论文目录
*   **代码组织：**
    *   `app/(site)/`：面向公众内容
    *   `app/(dashboard)/`：后台管理界面
*   **样式机制：**
    *   新组件使用 Tailwind + 局部样式
    *   Legacy 风格放在 `styles-legacy.css`，逐步拆分

### 后端（backend/）

*   **技术栈：** Express + Prisma + PostgreSQL
*   **功能模块：**
    *   邮箱 + 密码认证、JWT 鉴权
    *   文章 CRUD（后台用）
    *   时间线：`/api/timeline`（分页 / 搜索 / 排序）
    *   链接：`/api/links`
    *   论文：`/api/papers`
*   **数据脚本：**
    *   `npm run seed:timeline`
    *   `npm run seed:links`
    *   `npm run seed:papers`
    *   *均从 `frontend/public/data/*.json` 导入结构化内容*

### 会话与访问控制

*   登录态通过 React Context 注入，token 存储于 localStorage
*   后台 `/dashboard/*` 使用 middleware 保护
*   登录成功自动刷新用户状态与跳转

### Legacy 模块

*   静态站点存放于 `frontend/public/`（对外可访问）
*   `_legacy-static/` 作为历史备份，不再参与渲染逻辑
*   部分专题页尚未组件化，将在未来迁移到 CMS

## 🧠 五、长期愿景（Long-Term Vision）

*   **知识库化（Knowledge Graph Ready）**
    所有聚变相关内容结构化，为未来 AI/检索/推荐引擎提供基础。

*   **科研支持能力**
    支持科研人员查看实验路线、技术路线图、论文趋势、机构信息。

*   **AI 集成（Fusion Research Copilot）**
    包括：
    *   聚变概念解释与对比
    *   自动总结论文
    *   代码辅助（物理仿真、数据分析）
    *   自动构建路线图或技术演化图

*   **可视化与仿真展示**
    集成聚变装置结构图、技术流图、参数演化图等可视化能力。

**最终目标：** 形成一个能够长期迭代、协作、开放的数据与知识基础设施，支撑中国核聚变事业发展。
