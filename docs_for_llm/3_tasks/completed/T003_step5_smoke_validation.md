# T003 / Step 5 – 手动验证与收尾

## 目标
在不跑自动化测试的前提下，用明确的冒烟清单验证文章功能链路，并更新文档/日志收口。

## 主要产出
- 冒烟清单结论：默认列表、分页跳转、关键词、分类、标签、年份、多条件组合、空结果、加载更多/分页、刷新持久化；详情渲染/跳转/404。
- 已知风险与回滚策略：若 API 不兼容或数据缺失的应对方案。
- 文档更新：`T003_article_render.md` 与 `dev_notes.md` 的测试记录与状态更新。

## 执行步骤
1. 准备测试数据（必要时运行 seeds/fixtures），确认覆盖分类/标签/年份多样性。
2. 依次执行冒烟场景：  
   - 默认列表加载；  
   - 单条件筛选（关键词/分类/标签/年份）；  
   - 多条件组合；  
   - 空结果展示；  
   - 分页或加载更多；  
   - 刷新后筛选持久化；  
   - 详情页加载、Markdown 渲染、404。  
3. 记录每个场景的结果与异常，若遇到后端不支持的参数，按回滚/降级方案处理并记录。
4. 更新文档与日志：在 `T003_article_render.md` 标记验收状态，在 `dev_notes.md` 追加时间戳记录（包含验证范围、结论、剩余风险）。
5. 如需交付，准备摘要给团队（变更点、验证结论、待办/风险）。
